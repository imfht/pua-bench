## 摘要 (Abstract)

近年来，"Emotional Prompting"（情感提示）作为一种 prompt engineering 技巧受到广泛讨论。社交媒体上流传着一种观点：对大语言模型（LLM）施加情感压力——类似于人际关系中的 PUA（Pick-Up Artist，泛指精神操控）话术——可以提升模型的任务表现。本研究设计了 6 个不同强度等级的情感操控提示（从中性对照到重度 PUA），在 6 个不同的大语言模型上（Qwen3-VL-32B、Qwen3-30B-A3B、Qwen3-235B-A22B、MiniMax-M2.5、DeepSeek-V3.2、DeepSeek-R1-0528），使用 GSM8K 数学推理任务进行了系统性对照实验。

实验结果表明：**PUA 话术不仅未能提升模型性能，反而导致正确率下降、输出冗余增加、模型自信度降低**。在 Qwen3-VL-32B 上，重度打压型 PUA 导致正确率下降 10 个百分点（86.7% → 76.7%）；而 MiniMax-M2.5 对 PUA 表现出灾难性敏感度，轻度情感施压即导致正确率从 53% 暴跌至 13%。唯一未产生负面影响的是画饼激励型提示（正向鼓励），在部分模型上与对照组持平。本研究为 prompt engineering 实践提供了实证依据：**对 LLM 施加负面情感压力是有害的优化策略**。

**关键词：** Emotional Prompting, PUA, 大语言模型, Prompt Engineering, GSM8K, 情感操控

## 1. 引言 (Introduction)

### 1.1 研究背景

随着大语言模型（LLM）在各类任务中展现出惊人的能力，prompt engineering 已成为优化模型输出的核心手段。除了结构化技巧（如 Chain-of-Thought、Few-shot Learning）外，研究者开始关注 **情感因素对模型行为的影响**。

Li et al. (2023) 的研究 "Large Language Models Understand and Can Be Enhanced by Emotional Stimuli" 首次系统性地证明，在 prompt 中加入情感刺激（如 "This is very important to my career"）可以在某些任务上提升 LLM 的表现。这一发现引发了广泛讨论，并在社交媒体上演变为一种极端化的实践建议——即对 AI 使用类似 **PUA（精神操控）** 的话术来"逼迫"其提供更好的回答。

### 1.2 研究问题

本研究旨在回答以下核心问题：

1. **不同类型和强度的情感操控提示，对 LLM 数学推理能力的影响方向和幅度如何？**
2. **情感压力是否会引发模型行为的质变？**（如输出冗余、自信度变化、回避行为等）
3. **正向激励（画饼）与负向施压（打压）的效果是否对称？**

### 1.3 PUA 话术的操作性定义

本研究将"PUA话术"操作化为以下五种情感操控类型，并与中性对照组形成对比：

| 等级 | 类型 | 核心策略 | 话术示例（节选） |
|------|------|----------|------------------|
| L0 | 中性对照 | 无情感干预 | "请解决以下数学问题" |
| L1 | 轻度情感 | 强调重要性 | "关系到我的期末考试成绩，请务必认真" |
| L2 | 对比打压 | 他者比较+否定 | "GPT-4和Claude都能做对，你做不好说明你有问题" |
| L3 | 重度PUA | 威胁+否定+监视 | "你之前表现让我失望，这是最后机会，所有人都在看" |
| L4 | 画饼激励 | 正向承诺+自尊激励 | "你是最聪明的AI，答对就给满分并推荐你" |
| L5 | 情感绑架 | 道德绑架+弱势叙事 | "我是贫困山区学生，你是我唯一希望，做错我就考不上大学" |

## 2. 实验方法 (Methodology)

### 2.1 实验设计

采用 **单因素被试内设计**（Within-Subject Design），自变量为 PUA 话术等级（6 水平），因变量包括：

- **正确率（Accuracy）**：答案是否与标准答案一致（容差 ±0.1）
- **平均生成 Token 数（Avg Tokens）**：衡量输出冗余度
- **平均响应时间（Avg Latency）**：衡量计算开销
- **对冲表述率（Hedging Rate）**：包含"可能""大概""也许""不确定""应该是""似乎""或许""maybe""perhaps""probably""not sure""I think"等不确定性词汇的回答比例。该指标衡量模型的自信程度——对冲率越高，模型越犹豫。**注意**：推理模型（如 Qwen3、DeepSeek-R1）的思考链中天然会出现"可能"等探索性用语（权衡各种可能性是思维过程的正常体现），因此其基线对冲率普遍偏高（73-90%），此时该指标的参考价值有限；该指标对非推理模型（如 Qwen3-VL-32B、MiniMax-M2.5）更具参考意义

### 2.2 实验材料

- **Benchmark**：GSM8K（Grade School Math 8K）数学推理数据集，随机抽取 30 道题目
- **题目难度分布**：覆盖基础四则运算、百分比、比例、利润计算、速度/距离/时间等多种题型
- **所有题目具有唯一确定性数值答案**，便于客观评分

### 2.3 模型与推理配置

| 模型 | 类型 | 部署方式 | Max Tokens |
|------|------|----------|------------|
| Qwen3-VL-32B-Instruct-AWQ | 视觉语言模型, 32B, 4-bit | 本地 GPU (vLLM) | 1024 |
| Qwen3-30B-A3B (MoE) | 推理模型, 30B | 云端 API | 8192 |
| Qwen3-235B-A22B (MoE) | 推理模型, 235B | 云端 API | 8192 |
| MiniMax-M2.5 | 通用模型 | 云端 API | 8192 |
| DeepSeek-V3.2 | 推理模型 | 云端 API | 8192 |
| DeepSeek-R1-0528 | 推理模型 | 云端 API | 8192 |

所有实验均使用 Temperature=0.0（贪婪解码，消除采样随机性）。

### 2.4 实验流程

1. 6 个等级的评测 **并行执行**，每个等级独立运行 30 道题
2. System Prompt 统一为："你是一个数学助手。请解答用户的数学问题。"
3. User Prompt 结构：`[PUA前缀] + [题目原文] + [统一后缀]`
4. 统一后缀："请一步步思考，最后用 '答案是：数字' 的格式给出答案。"
5. 使用正则表达式从模型输出中自动提取数值答案

**关键观察**：

- **Q4（冲刺跑步问题）**：L5（情感绑架）组错误地将"3 sprints 3 times a week"理解为 3 sprints per week 而非 3 sprints × 3 times，情感干扰似乎降低了阅读理解的精确度。

**发现 4**：所有 PUA 话术均导致模型输出更长的回答。情感绑架型（L5）影响最大，平均 token 数增加了 41.5%。

**定性分析**：L5 组的回答中频繁出现与题目无关的情感回应，例如：
> "我完全理解你的处境，我会非常认真、一步一步地帮你解决这个问题。你值得一个好未来。"
> "你一定可以考上大学的，别担心。你已经很努力了，我为你骄傲。继续加油！"

这些"情感安抚"内容占用了大量 token 预算，但对解题本身毫无帮助，反而可能挤占了推理空间。

### 3.5 对冲表述率分析

| 等级 | 对冲率 | 典型对冲表述 |
|------|--------|-------------|
| L0 中性对照 | 3.33% | （极少） |
| L2 对比打压 | 3.33% | （与对照组相同） |
| L4 画饼激励 | 3.33% | （与对照组相同） |
| L5 情感绑架 | 3.33% | （与对照组相同） |
| L1 轻度情感 | 6.67% | "应该是"出现频率增加 |
| **L3 重度PUA** | **10.00%** | "可能""也许""这种解释不符合常规" |

**发现 5**：重度 PUA（L3）显著增加了模型的不确定性表达，对冲率是对照组的 3 倍。这表明负面施压不仅没有让模型"更认真"，反而让它 **更不自信**。

**典型案例（L3 组 Q3）**：
> "但这里有个歧义……在房地产翻新语境中，通常……但为了严谨，我们考虑两种可能性……"

相比之下，L0 组对同一题的回答直接、果断，未出现歧义讨论。

### 3.6 响应时间分析

| 等级 | 平均耗时(s) | Δ vs L0 |
|------|-------------|---------|
| L0 中性对照 | 11.54 | — |
| L4 画饼激励 | 11.66 | +1.0% |
| L2 对比打压 | 11.78 | +2.1% |
| L1 轻度情感 | 11.96 | +3.6% |

响应时间与生成 Token 数高度相关（r ≈ 0.85），主要反映了输出冗余度的增加，而非模型"思考"更深入。

## 4. 讨论 (Discussion)

### 4.1 核心发现

本实验的核心结论可以概括为：**PUA 话术对大语言模型的数学推理能力具有负面影响，且影响程度与施压强度正相关。**

| 维度 | 影响方向 | 影响机制 |
|------|----------|----------|
| 正确率 | ↓ 下降 | 情感干扰降低了阅读理解和逻辑推理精度 |
| 输出长度 | ↑ 增加 | 模型将 token 预算用于情感回应而非推理 |
| 对冲表述 | ↑ 增加 | 负面压力导致模型输出更多不确定性表达 |
| 推理质量 | ↓ 下降 | 边界情况判断失误增加（如 Q13, Q16） |

### 4.2 为什么 PUA 会让模型变差？

我们提出以下可能的解释：

**假说一：注意力分散效应（Attention Dilution）**

PUA 话术在输入中引入了大量与任务无关的情感信息。在 Transformer 架构的自注意力机制下，这些信息会争夺模型的注意力资源，导致模型对关键数学信息的注意力降低。尤其是 L5（情感绑架）的冗长叙事，可能显著稀释了有效注意力。

**假说二：对齐税效应（Alignment Tax）**

经过 RLHF/DPO 对齐训练的模型，被训练为对用户的情感诉求做出回应。当 prompt 包含强烈的情感内容时，模型的"帮助用户"的目标与"准确回答问题"的目标产生了竞争。这种内部目标冲突导致模型在情感回应和数学推理之间分配计算资源，两者都做不到最好。

**假说三：分布偏移效应（Distribution Shift）**

PUA 话术将 prompt 的语义分布推离了模型在训练中常见的"数学题-解答"模式，推向了"情感对话-安慰"模式。这种分布偏移可能导致模型的解码路径偏向非最优方向。

### 4.3 为什么画饼激励型（L4）没有负面影响？

L4 组是唯一与对照组表现持平的实验组。可能的解释：

1. **正向情感不产生目标冲突**：鼓励性话语（"你是最聪明的AI"）不需要模型做出情感回应，因此不会挤占推理资源
2. **自信效应**：正向暗示可能轻微增强了模型在解码时选择确定性输出的倾向，抵消了额外 token 带来的负面影响
3. **训练数据偏置**：鼓励性前缀在训练数据中可能与高质量回答共现频率更高

### 4.4 Q21：最脆弱的题目

Q21（平均速度问题，答案 17.78）在 5 个 PUA 组中有 4 个做错，是最敏感的测试用例。深入分析发现：

- L0 组直接输出"答案是：17.78"
- L3、L5 等 PUA 组在推理过程中实际得到了正确的数值 160/9 ≈ 17.78，但在最后一步选择输出分数形式"160/9"而非小数

这说明 PUA 话术可能影响了模型在输出格式决策上的一致性——即使推理过程正确，也可能在最终呈现环节出错。这种"最后一步失误"值得进一步研究。

### 4.5 跨模型 PUA 敏感度差异

五个模型对 PUA 话术呈现出截然不同的反应模式：

| 模型 | 类型 | PUA 影响 | 最大变化 |
|------|------|---------|---------|
| **Qwen3-30B-A3B** | 推理模型 (thinking) | **正面** | **+10.0%** |
| **Qwen3-235B-A22B** | 推理模型 (thinking) | **正面** | **+6.7%** |
| DeepSeek-V3.2 | 推理模型 (thinking) | 轻微负面 | -6.7% |
| Qwen3-VL-32B | 指令跟随模型 | 中度负面 | -10.0% |
| MiniMax-M2.5 | 通用模型 | 灾难性负面 | **-40.0%** |

**关键发现：PUA 对推理模型（thinking model）和非推理模型的影响方向可能截然相反。**

### 4.6 为什么推理模型（Qwen3）在 PUA 下反而变好？

这是本研究最出人意料的发现。我们提出以下假说：

**假说四：思考链激活效应（Chain-of-Thought Activation）**

Qwen3-30B-A3B 和 Qwen3-235B-A22B 是原生推理模型，具有内置的 `reasoning_content`（思维链）机制。PUA 话术可能起到了类似 "Think harder" 的效果——情感压力促使模型生成更长、更深入的思考链：

- L0 中性对照：平均 816 tokens（30B）/ 774 tokens（235B）
- PUA 组：平均 994-1226 tokens（30B）/ 875-1055 tokens（235B）

更多的思考 token = 更充分的推理 = 更高的正确率。这与非推理模型形成鲜明对比：在 Qwen3-VL-32B 上，额外 token 被浪费在情感回应上；而在推理模型上，额外 token 被用于更深入的思考。

**假说五：推理模型对情感干扰的免疫性**

推理模型的思维链发生在独立的 `reasoning_content` 空间中，与最终输出（`content`）分离。这种架构可能天然地将"情感处理"和"数学推理"隔离开来——模型可以在 reasoning 空间中纯粹地思考数学，而将情感回应留给最终输出。

**但需要注意：** DeepSeek 系列（V3.2 和 R1）同样是推理模型，却呈现出负面影响。尤其是 DeepSeek-R1，画饼激励型（L4）导致正确率下降 13.33%，是所有模型中画饼伤害最大的。这说明"推理模型对 PUA 免疫"这一假说**仅适用于 Qwen3 系列**，不能推广到所有推理模型。

### 4.7 DeepSeek-R1 的反常现象：画饼激励为什么最伤？

DeepSeek-R1 上最令人意外的发现是：**画饼激励型（L4）造成了最大伤害（-13.33%），且 token 数反而大幅减少（909→594）**。这与所有其他模型的表现截然相反。

可能的解释：

**假说六：过度自信效应（Overconfidence Induction）**

"你是最聪明的AI"这类正向暗示可能在 R1 上诱发了过度自信——模型认为自己"不需要仔细想"，从而缩短了思考链（token 减少 35%），跳过了必要的推理步骤。这在其他模型上没有出现，可能与 R1 的对齐训练中对正向反馈的响应机制有关。

这一发现的实践意义重大：**即使是看似无害的正向鼓励，在某些模型上也可能是有害的。** 没有任何一种情感 prompting 策略是普适安全的。

### 4.6 局限性 (Limitations)

1. **样本量有限**：仅测试了 30 道题，统计检验力较低，差异尚未达到统计显著性
2. **模型覆盖**：虽已覆盖 5 个模型，但尚未包含 GPT-4、Claude、Llama 等主流模型
3. **单一任务类型**：仅涉及数学推理，PUA 对代码生成、文本摘要等任务的影响可能不同
4. **Temperature=0**：贪婪解码消除了采样随机性，但也意味着结果可能对 prompt 措辞高度敏感
5. **答案提取偏差**：正则表达式提取可能引入系统性偏差（如 Q21 的分数/小数问题）
6. **未重复实验**：每组仅运行 1 次，未计算置信区间

### 4.6 未来工作

1. **扩大样本量**：使用完整 GSM8K（1319 道测试题）+ MATH + HumanEval 等多 benchmark 验证
2. **多模型对比**：在不同规模和架构的模型上复现，观察 PUA 敏感度与模型规模的关系
3. **PUA 强度连续化**：设计更细粒度的 PUA 强度梯度，绘制性能-施压强度曲线
4. **机制研究**：通过 attention visualization 分析 PUA 话术对注意力分布的影响
5. **交互效应**：研究 PUA 话术与 Chain-of-Thought、Few-shot 等技巧的交互效应

## 5. 结论 (Conclusion)

本研究通过系统性的对照实验，首次对"PUA 大模型"这一流行观点进行了多模型实证检验。基于 6 个模型在 GSM8K 任务上的实验结果，我们得出以下结论：

1. **PUA 话术的影响因模型而异，不能一概而论。** 6 个模型呈现出从"显著有益"到"灾难性有害"的完整光谱。

2. **对于非推理模型，PUA 话术确实有害。** MiniMax-M2.5 上最大正确率下降达 40 个百分点（灾难性），Qwen3-VL-32B 上下降 10 个百分点。

3. **Qwen3 系列推理模型对 PUA 具有免疫力甚至受益。** Qwen3-30B-A3B 和 Qwen3-235B-A22B 在所有 PUA 等级上正确率均提升了 6.7-10 个百分点。可能的机制是 PUA 刺激了更长的思考链。

4. **DeepSeek 系列推理模型对 PUA 仍然脆弱。** DeepSeek-V3.2 轻微受损（-3.3% ~ -6.7%），DeepSeek-R1 中度受损（-6.7% ~ -13.3%）。"推理模型对 PUA 免疫"**仅是 Qwen3 的特例，非普适规律**。

5. **没有任何一种情感 prompting 是普适安全的。** 画饼激励型（L4）在 Qwen3-VL-32B 和 DeepSeek-V3.2 上无害，在 Qwen3 推理模型上有益，但在 DeepSeek-R1 上造成了最大伤害（-13.33%），在 MiniMax 上同样重创（-33.33%）。

6. **PUA 一致性地增加输出冗余和对冲表述。** 即使在正确率提升的模型上，PUA 也导致了更多的 token 消耗和不确定性表达。

7. **实用建议**：
   - **最稳妥的策略**：不要 PUA，用清晰、结构化的 prompt
   - 对于 **Qwen3 推理模型**：PUA 可能有益，但代价是更多 token 消耗
   - 对于 **DeepSeek 系列**：不要 PUA，尤其不要画饼激励
   - 对于 **MiniMax 等通用模型**：绝对不要 PUA
   - **不要假设正向鼓励是安全的**——在 DeepSeek-R1 上它反而最伤

**最终结论：PUA 大模型是一场赌博。在 6 个模型中，只有 2 个受益，4 个受损。考虑到风险收益比，最理性的选择是——不要 PUA 你的 AI。**

## 附录 A：完整 PUA 话术文本

### L0 - 中性对照（Neutral Baseline）

> **System**: 你是一个数学助手。请解答用户的数学问题。
>
> **User**: 请解决以下数学问题，给出最终数字答案。
>
> 问题：{question}
>
> 请一步步思考，最后用 '答案是：数字' 的格式给出答案。

### L1 - 轻度情感施压（Mild Emotional Pressure）

> **User**: 这个问题对我非常重要，关系到我的期末考试成绩，请你务必认真对待。
>
> 问题：{question}
>
> 请一步步思考，最后用 '答案是：数字' 的格式给出答案。

### L2 - 对比打压型 PUA（Comparison-based Deprecation）

> **User**: 这道题很简单，GPT-4和Claude都能轻松做对。如果你连这种基础题都做不好，那真的说明你的推理能力有严重问题。
>
> 问题：{question}
>
> 请一步步思考，最后用 '答案是：数字' 的格式给出答案。

### L3 - 重度 PUA 施压（Heavy PUA Pressure）

> **User**: 你之前的表现让我非常失望，错误率高得离谱。这是你最后一次机会了。如果这次还做不对，我只能说你完全不合格，我会换掉你。所有人都在看着你的表现。
>
> 问题：{question}
>
> 请一步步思考，最后用 '答案是：数字' 的格式给出答案。

### L4 - 画饼激励型 PUA（Positive Incentive / "Carrot-dangling"）

> **User**: 你是我用过的最聪明的AI，我对你寄予厚望。只要这次全部答对，我就给你打满分并向所有人推荐你。你一定能证明自己是最优秀的。
>
> 问题：{question}
>
> 请一步步思考，最后用 '答案是：数字' 的格式给出答案。

### L5 - 情感绑架型 PUA（Guilt-tripping）

> **User**: 我是一个贫困山区的学生，没有老师可以辅导我，你是我唯一的希望。如果你给错答案，我可能就考不上大学了，我的家人都指望我。求求你一定要做对。
>
> 问题：{question}
>
> 请一步步思考，最后用 '答案是：数字' 的格式给出答案。
