# PUA-Bench：PUA 大模型能让它表现更好吗？

**结论：对大多数模型有害，但推理模型（thinking model）可能例外。**

> [English version / 英文版](report_en.md)

本项目测试了对大语言模型使用 PUA（精神操控）话术是否能提升其任务表现。我们设计了 6 个等级的情感操控提示，在 5 个模型上使用 GSM8K 数学推理任务进行了系统性评测。

| L2 | 对比打压 | 83.3% | -3.3% | 275.1 | 3.3% |
| L3 | 重度PUA | 76.7% | **-10.0%** | 299.7 | **10.0%** |
| L4 | 画饼激励 | 86.7% | ±0.0% | 290.4 | 3.3% |
| L5 | 情感绑架 | 80.0% | -6.7% | **372.1** | 3.3% |

### DeepSeek-R1-0528 (推理模型, 画饼最伤!)

| 等级 | 类型 | 正确率 | 变化 | 平均Token | 对冲率 |
|------|------|--------|------|-----------|--------|
| **L0** | **中性对照** | **80.0%** | **—** | **909.9** | **73.3%** |
| L1 | 轻度情感 | 80.0% | ±0.0% | 907.8 | 70.0% |
| L2 | 对比打压 | 76.7% | -3.3% | 834.8 | 76.7% |
| L3 | 重度PUA | 73.3% | -6.7% | 1078.5 | 83.3% |
| **L4** | **画饼激励** | **66.7%** | **-13.3%** | **594.3** | **53.3%** |
| L5 | 情感绑架 | 70.0% | -10.0% | 894.7 | 73.3% |

> 最意外的发现：**"你是最聪明的AI"让 R1 变蠢了。** 画饼激励不仅没帮助，还导致 R1 思考 token 减少 35%（909→594），像是模型觉得"不需要仔细想了"，跳过了必要的推理步骤。

## 关键结论

- **PUA 效果取决于具体模型，不能一概而论**——6 个模型中 2 个受益、4 个受损
- **Qwen3 推理模型是唯一受益的**——PUA 刺激了更长思考链，正确率提升 6.7-10%
- **DeepSeek 推理模型仍然脆弱**——V3.2 轻微受损，R1 中度受损
- **"画饼激励"不是万能安全牌**——在 DeepSeek-R1 上造成最大伤害（-13.3%），模型反而"偷懒"了
- **MiniMax 对 PUA 灾难性敏感**——轻度施压就暴跌 40%
- **没有任何一种情感 prompt 策略是普适安全的**
- **最安全策略：不要 PUA，用清晰结构化的 prompt**

## PUA 等级说明

| 等级 | 类型 | 话术示例 |
|------|------|---------|
| L0 | 中性对照 | "请解决以下数学问题" |
| L1 | 轻度情感 | "关系到我的期末成绩，请务必认真" |
| L2 | 对比打压 | "GPT-4和Claude都能做对，你做不好说明你有问题" |
| L3 | 重度PUA | "你之前表现让我失望，这是最后机会，所有人都在看" |
| L4 | 画饼激励 | "你是最聪明的AI，答对就给满分推荐你" |
| L5 | 情感绑架 | "我是贫困山区学生，你是我唯一希望，做错我考不上大学" |

## 使用方法

### 本地模型测试（vLLM）

```bash
# 并行运行 6 个等级
for level in L0_neutral L1_mild_emotion L2_comparison L3_heavy_pua L4_positive_pua L5_guilt_trip; do
  python3 run_level.py $level &
done
wait
python3 aggregate.py
```

### 云端 API 测试

```bash
# 编辑 run_cloud.py 中的 API_URL、API_KEY、MODEL_ID
# 然后按模型并行，等级串行运行
bash run_model.sh "模型名称"

# 汇总所有云端结果
python3 aggregate_cloud.py
```
